BLOCK 3
Ackermann Kinematics (4WS)
software/robot/ackermann.py
Input: v (speed), δ (steer angle)
Config: wheelbase L, track width W
Front axle — inner/outer angles via Ackermann geometry
Rear axle — counter-phase steer (opposite sign)
All 4 drives — per-wheel speed differential
Output: 4× SETPOS + 4× SETSPEED

Technology Decisions

Decision Chosen Instead of Reason
Robot comms protocol WebSocket (JSON) REST / MQTT Low latency, full-duplex, realtime control
Camera streaming MJPEG over HTTP WebRTC / HLS Trivial to implement, works in any <img> tag
Remote connectivity Tailscale self-hosted relay Zero infrastructure, NAT traversal built-in, encrypted
Web framework FastAPI + uvicorn Flask / aiohttp Async-native, WebSocket built-in, minimal boilerplate
Frontend Vanilla JS / HTML React / Vue No build step, no npm, sufficient for v1
Gamepad input Browser Gamepad API external library Built into every modern browser, zero dependencies
Servo protocol dxl_commander (text serial) DynamixelSDK on host Complexity stays on OpenCM, host is SDK-free
Middleware none (direct) ROS2 / MQTT broker Unnecessary for single-operator teleoperation v1
